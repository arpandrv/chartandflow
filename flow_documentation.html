Conversation Flow

FSM State Transitions

Note: "2nd Attempt" happens when the user response is unclear or is against the current FSM state. The system will try once more to get a relevant response before gracefully moving forward.

stateDiagram-v2 [*] --> Welcome: User Starts Welcome --> SupportPeople: Ready/Continue Welcome --> Welcome: 2nd Attempt SupportPeople --> Strengths: Has Support SupportPeople --> SupportPeople: 2nd Attempt Strengths --> Worries: Shares Strengths Strengths --> Strengths: 2nd Attempt Worries --> Goals: Continue Discussion Worries --> Worries: 2nd Attempt Goals --> LLMConversation: Sets Goals Goals --> Goals: 2nd Attempt LLMConversation --> LLMConversation: Free Chat LLMConversation --> [*]: End Session

4-Step Model

  1. 1
    Support People: Identify personal support network
  2. 2
    Strengths: Recognize personal strengths and capabilities
  3. 3
    Worries: Discuss concerns with risk monitoring
  4. 4
    Goals: Set achievable future objectives

NLP Processing

Intent Classification:

Zero-shot BART → LLM fallback

Sentiment Analysis:

Twitter-RoBERTa → LLM fallback

Risk Detection:

LLM classifier → HF suicidality model

Response Selection:

Sentiment-aware, culturally appropriate

Response Pool Selection

  • 📂 responses.json categories
  • 🎯 Current FSM state filter
  • 😊 Sentiment-aware variation
  • 🎪 Intent-specific responses
  • 🔄 Transition prompts
  • 🌏 Culturally appropriate language

Clarification Strategy

  • ❓ State-specific clarifying questions
  • 🎯 Guide user toward expected response
  • 🔄 Maximum 1 retry attempt
  • ✅ Graceful acknowledgment on failure
  • ⏭️ Continue conversation flow
  • 🔒 Preserve UX fluidity

LLM Handoff Mode

  • 🔄 After 4 FSM steps complete
  • 💬 Full conversation context
  • 🎭 Personality consistency
  • 🚨 Continued risk monitoring
  • 📝 Session persistence
  • ∞ Open-ended conversation
